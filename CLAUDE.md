# 3분 뒤 보스전! - 개발 지침서

이 문서는 Claude 에이전트가 이 프로젝트를 이어서 작업할 때 참고하는 지침서입니다.

---

## 프로젝트 개요

"3분 뒤 보스전!"은 2D 플랫포머 탐색 + 반복형 스토리 게임입니다. 3분 안에 5개의 키 아이템을 찾아야 하며, 실패 시 이길 수 없는 보스가 등장하여 강제 사망 후 반복합니다. 반복을 통해 스토리 조각이 공개되는 구조입니다.

상세 기획서: `3분_뒤_보스전_기획서.md` 참고

---

## 기술 스택

- **엔진:** Phaser.js 3 (CDN으로 로드)
- **언어:** JavaScript (ES6+)
- **배포:** GitHub Pages (아이폰 사파리에서 실시간 확인 목적)
- **목표 플랫폼:** 모바일 (iOS 우선), PC 확장 가능
- **향후 확장:** Capacitor(모바일 앱), Electron(스팀) 등 검토 예정

---

## 팀 구성 & 역할

- **덕영:** 기획, 디렉팅, 의사결정
- **Claude:** 코드 개발, 구조 설계, 문서 작성
- **그래픽:** AI 생성 (디자이너 없음, 프로토타입은 단순 도형/실루엣으로 진행)
- **사운드:** 별도 구매 또는 무료 소스 활용 (프로토타입에서는 생략 가능)

---

## 현재 단계

### 프로토타입 (1스테이지 핵심 루프)

구현해야 할 것:

1. **플레이어 이동** - 2D 플랫포머 (좌/우 이동, 점프, 낙하, 중력)
2. **맵** - 플랫폼 기반 탐색 맵 (타일맵 또는 수동 배치)
3. **타이머** - 180초 카운트다운, 화면 상단 표시
4. **아이템** - 맵에 5개 배치, 접촉 시 수집
5. **보스 등장** - 타이머 0초 도달 시 보스 등장 연출
6. **사망 & 반복** - 보스 등장 후 강제 사망 → 스테이지 처음으로
7. **클리어** - 5개 모두 수집 시 스테이지 클리어 처리

### 프로토타입에서 생략 가능한 것

- 스토리 연출 / 대사
- 사운드 / BGM
- 정교한 그래픽 (단색 도형으로 충분)
- 노멀/레드 루트 분기
- 인트로 / 엔딩 씬

## 코딩 컨벤션

- Phaser 3 Scene 기반 구조 사용
- 각 씬은 별도 파일로 분리
- 변수/함수명: camelCase
- 상수: UPPER_SNAKE_CASE
- 한글 주석 사용 (덕영 님이 코드를 읽을 수 있도록)
- 모바일 터치 입력 고려 (터치 + 키보드 모두 지원)

## 배포 방법

1. GitHub 레포지토리: `MeetBossAfter3Mins`
2. GitHub Pages 활성화 (main 브랜치 / root)
3. push 시 자동 배포
4. 아이폰 사파리에서 `https://[username].github.io/MeetBossAfter3Mins/` 접속하여 확인

## 주의사항

- 덕영 님은 디자이너가 아니므로, 그래픽 관련 작업은 AI 생성 에셋을 활용하거나 프로토타입 단계에서는 placeholder 사용
- 개발 엔진 최종 결정은 미정 (Phaser.js로 프로토타입 후 판단)
- 모바일 우선이므로 터치 조작, 화면 비율(세로/가로) 항상 고려
- 코드 수정 후 GitHub Pages로 바로 확인할 수 있는 구조 유지
